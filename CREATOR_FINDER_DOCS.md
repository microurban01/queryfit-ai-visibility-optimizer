
# Creator Finder (YouTube): Feature Documentation

## 1. Executive Summary
The **Creator Finder** module is QueryFit's "Active Promotion" engine. While the core platform focuses on *monitoring* AI visibility, the Creator Finder helps users *influence* it by discovering YouTube influencers who can generate high-authority citations.

It acts as a vertical search engine specifically tuned for **Influencer Discovery**, bypassing standard YouTube search noise to find channels that match specific subscriber "Sweet Spots" and engagement criteria.

## 2. Core Workflows

### A. Discovery & Filtering
The search engine uses a "Fetch & Filter" architecture. Because the YouTube API provides limited native filtering, the application fetches batches of results and applies strict client-side logic to ensure quality.

**Key Filters:**
*   **Channel Size Presets:** Micro (10k-50k), Mid (50k-250k), Macro (250k-1M), Mega (1M+).
*   **Geography & Language:** Filters by ISO country codes and content language.
*   **Content Safety:** "Exclude Kids Content" toggle (crucial for B2B brands).
*   **Activity:** "Active Only" toggle filters out ghost channels with 0 videos.

### B. The "Fit Score" Algorithm
To help users prioritize, every channel is assigned a proprietary **Fit Score (0-100)** calculated in real-time within `CreatorsYouTube.tsx`.

**Formula:**
1.  **Baseline:** Start at 50.
2.  **Engagement Bonus (+15):** If `(Avg Views / Subscribers) >= 10%`.
3.  **Consistency Bonus (+10):** If `Video Count > 50`.
4.  **Inactivity Penalty (-10):** If `Video Count < 10`.
5.  **"Sweet Spot" Bonus (+10):** If `Subscribers` are between 10k and 250k (High ROI zone).

### C. Deep Analysis (Influencer Report)
Clicking "View Analytics" opens an overlay generated by `YouTubeService.getInfluencerReport`.
*   **Engagement Rate:** Calculated as `(Likes + Comments) / Views`.
*   **Audience Demographics:** (Simulated/Predicted) breakdown of Age, Gender, and Geography.
*   **Content Health:** Recent video performance metrics compared to channel averages.

### D. List Management
*   **Shortlisting:** Users can "Heart" channels to save them to a local shortlist.
*   **Bulk Actions:** In Table View, users can multi-select channels to:
    *   Bulk Shortlist.
    *   **Export to CSV:** Generates a downloadable file containing Channel Name, URL, Subs, Email (if public), and Engagement stats.

## 3. Technical Architecture

### API Strategy: "Bring Your Own Key" (BYOK)
To circumvent the strict quotas of the YouTube Data API v3 (10,000 units/day default), the module requires the user to input their own API Key in the Settings or directly on the page.
*   **Storage:** Keys are stored in `integrationSettings` (local state/context), never sent to QueryFit servers.
*   **Endpoint:** Direct calls to `https://www.googleapis.com/youtube/v3`.

### Quota Management & Pagination
The search logic implements a **"Fetch Until Full"** strategy to handle the discrepancy between API results and User Filters.

1.  **The Problem:** Asking YouTube for "Tech" might return 50 results, but only 5 match the user's "Germany + >100k Subs" filter.
2.  **The Solution:** The app enters a `while` loop, fetching pages of 50 results sequentially until:
    *   It finds the target number of valid channels (e.g., 60).
    *   It hits a safety limit of API calls (e.g., 8 calls) to prevent quota exhaustion.
3.  **Error Handling:** If a `403 Quota Exceeded` error occurs mid-loop, the app gracefully degrades, showing whatever results have been collected so far and alerting the user.

### State Management
*   **`viewMode`:** Persisted in `localStorage` (`grid` vs `table`).
*   **`searchProgress`:** Tracks the number of raw items processed during the filtering loop to give UI feedback.
*   **`shortlistedChannels`:** Maintained in component state (in a production app, this would sync to a database).

## 4. UI/UX Design System

### Visual Language
*   **Platform Branding:** Unlike the rest of the app (Deep Amethyst), this page uses **Red/Carbon** accents to visually signify the "YouTube" context.
*   **Dual Layout:**
    *   **Grid View:** Highly visual cards focusing on thumbnails and quick stats. Best for browsing.
    *   **Table View:** Data-dense rows with sortable columns. Best for analysis and bulk operations.

### Components
*   **`CreatorsYouTube.tsx`:** The monolith controller for the page.
*   **`YouTubeService.ts`:** Handles API calls, response parsing, and report generation.
*   **`InfluencerReport` Overlay:** A modal dashboard analyzing specific channel metrics.

## 5. Strategic Value
This tool turns QueryFit from a passive monitoring tool into an active marketing tool. By finding creators who influence AI training data (via their video transcripts and descriptions), users can organically improve their AI visibility.
